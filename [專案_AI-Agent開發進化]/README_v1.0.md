\### 一、 專案目標 (Project Mission)

將原本單打獨鬥的 Vibe Coding (Antigravity + Gemini 3.1) 進化為「多 Agent 協作模式」，解決複雜系統開發時的功能斷點與前端整合問題。



\### 二、 第一階段：NotebookLM 知識庫建構 (當前重點)

在啟動 Coding 前，先建立 NotebookLM 專題筆記本，餵食以下內容：

1\. \*\*YouTube 案例：\*\* 搜尋 "AI Agent Coder Workflow", "Cursor Agent Mode best practice", "Vibe Coding Complex App"。

2\. \*\*GitHub 案例：\*\* 收集 open-source agent frameworks (如 LangGraph 範例)、高品質的 React/Tailwind 前端組件模板。

3\. \*\*技術文件：\*\* Gemini 3.1 API 限制說明、Antigravity 操作手冊。



\### 三、 第二階段：Agent 團隊架構定義 (WBS)

\- \*\*\[Agent-A] 指揮官 (Architect)：\*\* 負責拆解需求，定義 API 規格與模組邊界。

\- \*\*\[Agent-B] 執行工兵 (Coder)：\*\* 負責撰寫實體程式碼與單元測試。

\- \*\*\[Agent-C] 審查官 (Validator)：\*\* 負責迴歸測試，確保新功能不破壞舊功能。
